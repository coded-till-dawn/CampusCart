❖ PROBLEM STATEMENT
================================================================================

Campus-based peer-to-peer marketplace platforms are essential in modern 
educational institutions. However, many students lack a centralized platform 
to buy, sell, and exchange used items such as textbooks, electronics, furniture, 
and clothing within their campus community.

Key Challenges Addressed:

1. DISCOVERABILITY:
   - Students struggle to find used items available within their campus
   - No centralized marketplace for student-to-student transactions
   - Limited visibility into product availability and pricing

2. TRUST & SAFETY:
   - No verified seller profiles or buyer protection mechanisms
   - Risk of fraudulent transactions or product disputes
   - Lack of communication channels between buyer and seller

3. INVENTORY MANAGEMENT:
   - No centralized database for tracking available items
   - Difficulty in categorizing products (books, electronics, furniture, etc.)
   - No wishlist or saved items feature for later purchases

4. USER EXPERIENCE:
   - Fragmented solutions (WhatsApp groups, physical notices)
   - No search, filter, or sort functionality
   - Limited product information and images

Objectives of CampusCart:

✓ Create a unified, user-friendly online marketplace for campus community
✓ Enable simplified buying/selling of used college essentials
✓ Provide product categorization and advanced search capabilities
✓ Facilitate direct communication between buyers and sellers
✓ Implement a wishlist system for interested buyers
✓ Display products with detailed information and high-quality images

DATABASE SCHEMA:

Items Collection:
{
  _id: ObjectId,
  title: String,
  price: Number,
  category: String,
  description: String,
  image: String (URL),
  sellerName: String,
  contact: String,
  location: String,
  createdAt: Date,
  updatedAt: Date
}

Wishlists Collection:
{
  _id: ObjectId,
  itemId: ObjectId (ref: Items),
  addedAt: Date
}

Messages Collection:
{
  _id: ObjectId,
  itemId: ObjectId (ref: Items),
  senderName: String,
  senderContact: String,
  message: String,
  createdAt: Date
}


❖ CODING PART
================================================================================

KEY CODE SNIPPETS:

1. BACKEND - Express Server Setup (server.js):
   - Express app initialization
   - MongoDB connection with SSL
   - CORS configuration
   - Route mounting
   - Error handling middleware
   - Server listening on port 5000

2. FRONTEND - Product Display (app.js):
   - Fetches products from /api/items
   - Renders products in grid layout
   - Implements search and filter
   - Handles category selection
   - Wishlist interaction

3. FRONTEND - Image Fallback System (utils.js):
   - Automatic image URL error handling
   - 2-stage fallback: Unsplash → LoremFlickr
   - Generates relevant product-specific images
   - Prevents broken image icons

4. API LAYER - HTTP Communication (api.js):
   - axios-based HTTP client
   - RESTful endpoint definitions
   - Error handling and logging
   - Response parsing

5. DATABASE - MongoDB Connection (config/db.js):
   - Atlas SSL configuration
   - Connection timeout settings
   - Certificate validation settings
   - Error logging

Frontend:

INSTALLATION & SETUP:

1. Clone Repository:
   git clone [repository-url]
   cd CampusCart

2. Backend Setup:
   cd backend
   npm install
   Create .env file with MongoDB URI
   npm run seed (Load sample data)
   npm start (Run server)

3. Frontend Setup:
   cd frontend
   python -m http.server 8000

4. Access Application:
   Open http://localhost:8000 in browser

Environment Variables (.env):
   MONGODB_URI=mongodb+srv://user:pass@cluster.mongodb.net/campuscart
   PORT=5000
